### 年增长率公式修复（与Excel完全匹配）

**问题**：在线演算器的"生存总利益年增长率"与Excel原版不一致
- 旧公式：`rate = (total / prevTotal - 1) * 100`（同比增长率）
- Excel公式：`S = (O_current - O_prev) / E`（年增值÷累计保险费）

**Excel结构分析**：
- R列（年增值）= O_current - O_prev（本年生存总利益 - 上年生存总利益）
- S列（年增值率）= R / E（年增值 ÷ 累计保险费）
- 储备期逻辑：第1年"储备期"，第2~payYears+2年空白，第payYears+3年起显示增长率

**代码修改**：
- 公式改为 `((total - prevTotal) / cumPremiumAll) * 100`
- 储备期：yr=1→"储备期"，yr<=payYears+2→"-"，yr>=payYears+3→计算增长率
- 修复display mode过滤时 prevTotal 和 cumPremiumAll 不更新的bug（重构循环结构）

**验证结果**：
- 40岁女性3年交100万：65行数据全部匹配 ✓
- 4种不同保费（5万/20万/50万/200万）：全部匹配 ✓
- 公式与保费无关（premium在除法中抵消）

### 生存总利益计算公式根本性修复（K/L/N递归公式）

**问题根因**：CALC_DATA存储了数据表col P(16)交清增额现金价值系数，但错误地用 `paidup = P * scale` 计算累计红利现价。Excel实际用递归公式：
- K(年度交清增额) = ROUND((M*份数*10000 + L_prev*O_prev*0.01225) / O, 0)
- L(累计K) = SUM(K:K)
- N(累计红利现价) = ROUND(L * P / 10000, 0)

**修复内容**：
1. CALC_DATA从[J, P]改为[J, M*100, O, P]四列（M乘100避免浮点）
2. generate()和downloadImage()都改为递归计算K/L/N
3. 储备期改为前paymentYears+2年全部显示"储备期"（不再只有第1年）
4. 储备期单元格用rowspan合并居中
5. copyTable()处理rowspan导致的缺失单元格
6. Canvas下载图片中储备期也垂直居中合并显示

**验证结果**：
- 40岁女性3年交100万：20年数据(G/N/O/Rate)全部与Excel完全匹配 ✓
- 10组随机数据(不同性别/年龄/缴费期限/保费)：全部通过 ✓
